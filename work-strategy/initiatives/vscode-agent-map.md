# VSCode Agent Map

## What It Is
A Claude-based orchestration layer inside VSCode that allows users to stitch together multiple MCP-connected capabilities into unified workflows.

## Primary Users
- Cross-functional: product, engineers, analysts

## Core Job (Analysts)
- Streamline workflows
- Research across systems (ADO, Slack, diagrams)
- Automate workflow creation and maintenance

## Team Role
- Mid-late adopters
- Catching up to early adopter teams
- Future: Analytics Center of Excellence

## Current State
- Some teams ahead but with narrow use cases
- Tool capability broader than current usage

## Current Friction
- Tool fatigue
- Newness of tool
- Uneven team bandwidth

## Success (3–6 Months)
- Everyone has installed tool
- Everyone uses it in some capacity
- Team understands capabilities

## Long-Term Goal
- Team becomes CoE for Analytics
- Team develops deep expertise

## Explicit Non-Goals / Boundaries
- Team does not own tool
- Team is not yet expert
- Not all users need to be power users immediately
- Power user expectation is long-term, not immediate
